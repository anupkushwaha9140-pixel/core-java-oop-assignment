Java Assignment – User Story Based Implementation
Introduction
This assignment demonstrates core Java programming concepts through 8 user stories. Each story represents a practical requirement and is solved using fundamental Object‑Oriented Programming (OOP) principles, constructors, encapsulation, static variables, arrays, strings, and polymorphism.
The purpose is to connect theory with practice — showing how Java features are applied to real‑world scenarios like managing students, employees, products, and billing systems.
Implemented User Stories US‑01: Student Object Requirement: Create a Student class with rollNo, name, marks and display details.
Concepts: Class, Object, Fields, Methods
Deliverable: Student object created and details displayed successfully. US‑02: String Formatting Requirement: Format student name in original, uppercase, lowercase, and length.
Concepts: Strings, String methods (length(), toUpperCase(), toLowerCase())
Deliverable: Name formatted correctly using String methods. US‑03: Array of Marks Requirement: Store marks of 5 students and display them one by one.
Concepts: Arrays, for loop, Indexing
Deliverable: All marks displayed one by one. US‑04: Product Billing (Method Overloading) Requirement: Calculate bill for 1, 2, and 3 products.
Concepts: Methods, Parameters, Method Overloading, Polymorphism
Deliverable: Bill calculated for different number of products. US‑05: Employee Initialization (Constructor) Requirement: Initialize employee/trainer details directly.
Concepts: Constructor, Object Initialization
Deliverable: Employee object initialized directly and displayed. US‑06: Shared Company Name (Static Variable) Requirement: Use same company name for all employees.
Concepts: static Keyword, Shared Data
Deliverable: Same company name displayed for multiple employee objects. US‑07: Product Price Protection (Encapsulation) Requirement: Protect product price details using private variables.
Concepts: Encapsulation, private, Getter, Setter
Deliverable: Product details accessed securely using methods. US‑08: General vs College Students (Inheritance) Requirement: Distinguish between general students and college students.
Concepts: Inheritance, Parent Class, Child Class
Deliverable: Parent and child class properties displayed correctly.
🔹 Key Concepts Demonstrated Encapsulation → Protect sensitive data using private fields and controlled access.
Inheritance → Reuse common details across parent and child classes.
Polymorphism → Method overloading for flexible billing calculations.
Constructor → Direct object initialization for cleaner code.
Static keyword → Shared data across multiple objects.
Strings & Arrays → Efficient handling of text and collections of values.
🔹 Importance of This Assignment Builds a strong foundation in OOP principles.
Shows how Java features solve real‑world problems.
Prepares for college exams, interviews, and projects by combining theory with practical coding.
Demonstrates professional coding practices like encapsulation, constructors, and shared data. ======================================================================================================== 🔹 Expected Outcomes Each user story runs successfully with correct output.
